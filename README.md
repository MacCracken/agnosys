# Agnosys

> **Agnosys** (agnos + sys — toward knowledge of the system) — AGNOS kernel interface

Cyrius bindings for Linux kernel syscalls and security primitives. Consumers include only the modules they need, or include the single `dist/agnosys.cyr` bundle for the full surface area. The foundation layer that every AGNOS program depends on.

**1.2.5 · Cyrius 5.10.44 · 20 modules · 730 public fns · ~10 300 src lines · 153 KB binary (DCE) · ~170 ms compile · 0 runtime deps**

## Architecture

```
Consumer programs:
  yukti      ──→ agnosys[udev]              (device hotplug)        ✅
  kavach     ──→ agnosys[security]           (sandboxing)           ✅
  nein       ──→ agnosys[netns]             (network namespaces)    ✅
  stiva      ──→ agnosys[luks,dmverity]     (encrypted storage)     ✅
  sigil      ──→ agnosys[tpm,ima,certpin]   (trust verification)    ✅
  soorat     ──→ agnosys[drm]              (GPU rendering)          ✅
  libro      ──→ agnosys[audit]            (kernel audit)           ✅
  argonaut   ──→ agnosys[journald,bootloader] (init system)         ✅
  shakti     ──→ agnosys[pam]              (authentication)         ✅
  aegis      ──→ agnosys[mac]             (mandatory access control) ✅
  ark        ──→ agnosys[fuse,update]      (package management)     ✅
  daimon     ──→ agnosys[security,certpin]  (daemon runtime)        ✅
```

## Build

```sh
cyrius build src/main.cyr build/agnosys   # compile
cyrius run src/main.cyr                   # compile + run
cyrius check src/main.cyr                 # syntax check only
cyrius test                               # run tests/tcyr/*.tcyr
cyrius distlib                            # regenerate dist/agnosys.cyr (full)
cyrius distlib core                       # regen dist/agnosys-core.cyr
                                          # (also: security, storage, trust, system)
scripts/audit.sh                          # full local quality run (11 gates, mirrors CI)
```

## Consumer quickstart

Two consumption patterns. Pick one based on how much of agnosys you need.

### Per-module (feature-gated, smaller binary)

Consumers that need a specific slice pull individual modules:

```cyrius
#define LINUX
include "lib/alloc.cyr"
include "lib/tagged.cyr"
include "lib/syscalls.cyr"
include "src/error.cyr"
include "src/security.cyr"     # landlock, seccomp, namespaces

fn main() {
    alloc_init();
    # ... call agnosys functions ...
    return 0;
}
```

### Profile bundle (V1.2.0+, single include for one domain)

Slim profile bundles ship alongside the full one — pull just the domain you wire to. Cuts consumer-side distribution size 60-78% vs the full bundle:

| Bundle | Modules | Size |
|---|---|---|
| `dist/agnosys-core.cyr` | error + syscall + logging + arch peers | ~23 KB |
| `dist/agnosys-security.cyr` | security + mac + audit + pam | ~76 KB |
| `dist/agnosys-storage.cyr` | luks + dmverity + fuse | ~49 KB |
| `dist/agnosys-trust.cyr` | tpm + ima + secureboot + certpin | ~70 KB |
| `dist/agnosys-system.cyr` | journald + bootloader + udev + drm + netns + update | ~111 KB |

```cyrius
#define LINUX
include "dist/agnosys-core.cyr"      # error + syscall + logging
include "dist/agnosys-security.cyr"  # the slice you actually use
```

### Full bundle (single include, all 20 modules)

Shipped as `dist/agnosys.cyr` (~329 KB) — a concatenation of every module in declared
order, generated by `cyrius distlib`. One include, full surface area:

```cyrius
#define LINUX
include "dist/agnosys.cyr"     # all 20 modules
```

See [`docs/development/capacity-baseline.md`](docs/development/capacity-baseline.md) for the full compiler-table footprint.

## Quality gates

Every change must pass `scripts/audit.sh` — 11 gates, identical to CI:

```
syntax → API surface (snapshot + prose) → capability map → capacity →
build → smoke → tests → lint → vet → fuzz → benchmarks
```

| Gate | Command |
|---|---|
| API surface diff | [`scripts/check-api-surface.sh`](scripts/check-api-surface.sh) |
| API surface prose staleness | [`scripts/gen-api-surface-prose.sh`](scripts/gen-api-surface-prose.sh) `--check` |
| Capability map staleness | [`scripts/gen-capability-map.sh`](scripts/gen-capability-map.sh) `--check` |
| Capacity (<85% all tables) | `cyrius capacity --check src/main.cyr` |
| Tests (247 assertions) | `cyrius test` |
| Fuzz (7 harnesses) | `fuzz/*.fcyr` under `timeout 10 ... 500` |
| Benchmarks (30 across 11 groups) | `tests/bcyr/bench_all.bcyr` |
